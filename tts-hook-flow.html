Key Points

File Locations

  • ~/.claude/hooks/eleven_speak_queue.py — Hook script
  • ~/.claude/settings.json — Env vars + hook config
  • ~/.claude/tts_queue/jobs/ — Pending job spool
  • ~/.claude/tts_queue/audio/ — Temp MP3 files
  • ~/.claude/tts_queue/failed/ — Failed jobs with errors
  • ~/.claude/tts_queue/queue.lock — Global playback lock

Recursion Prevention

  • When the hook calls claude -p for summarization, it sets CLAUDE_TTS_INTERNAL=1
  • The nested Claude process fires its own Stop hook, but the script exits immediately when it sees that env var
  • Additionally, stop_hook_active from Claude Code prevents re-entrant hook loops

Queue Design

  • Jobs are written to the spool directory before acquiring the lock
  • Summarization happens before enqueueing (outside the lock) to avoid holding it during slow API calls
  • The lock holder drains all pending jobs (including ones from other sessions) before releasing
  • Failed jobs are moved aside so the queue never gets stuck

Text Pipeline

  • Raw message → remove code fences, markdown → truncate to 4000 chars (summary input)
  • If summarizing: → claude -p Haiku → N-sentence prose
  • Final speech text → sanitize again → truncate to 900 chars → send to ElevenLabs
